# Архитектурный план сайта отеля

## 1. Цель

Определить структуру проекта, разделение на страницы и компоненты, способы хранения данных и технологические слои для реализации PRD.

## 2. Общая архитектура

- Сайт ориентирован на мобильный, responsive-first интерфейс.
- Основа — статический фронтенд с возможностью подключения виджета бронирования.
- Контент организован через структурированные данные: номера, предложения, отзывы, контакты.
- Вёрстка строится на повторно используемых блоках/компонентах.

## 3. Страницы и маршруты

1. `/` — Главная
2. `/rooms.html` — Номера
3. `/services.html` — Услуги / Опыт
4. `/gallery.html` — Галерея
5. `/offers.html` — Акции
6. `/about.html` — О нас
7. `/contact.html` — Контакты
8. `/booking.html` — Бронирование / availability

## 4. Компоненты интерфейса

### 4.1. Общие компоненты

- Header / навигация с логотипом, меню и кнопкой CTA.
- Footer с контактами, быстрыми ссылками, соцсетями.
- Hero-блок с изображением/видео и основным CTA.
- Grid/карточки для номеров и предложений.
- Карта и контактный блок.
- Блок отзывов и соцдоказательств.

### 4.2. Специальные компоненты

- Booking widget: поля заезда, выезда, гостей и кнопка поиска.
- Benefits strip: короткие преимущества отеля.
- Image gallery / lightbox.
- Promo card / акции.
- Timeline / story section для страницы "О нас".

## 5. Данные и контент

### 5.1. Структуры данных

- Room:
  - id
  - name
  - description
  - price
  - capacity
  - amenities
  - image
  - link

- Offer:
  - id
  - title
  - description
  - period
  - CTA

- Testimonial:
  - author
  - rating
  - text
  - role / location

- Service:
  - id
  - title
  - description
  - icon / image

- Contact:
  - address
  - phone
  - email
  - map embed
  - social links

### 5.2. Хранение данных

- JSON / YAML / JS-объекты в `data/`.
- Контентные данные отделены от шаблонов.
- Изображения в `assets/images/`.
- Видео/медиа в `assets/media/`.

## 7. Технологический стек

- HTML5 + CSS3 + JavaScript.
- CSS utilities: custom properties, flexbox и grid.
- Progressive enhancement: статическое содержимое доступно без JS.
- Опционально: библиотека для lightbox / карусели.
- Опционально: статический сайт на Vite/Eleventy или простая сборка на npm.

## 8. Интеграция бронирования

- Форму поиска дат и гостей можно реализовать локально.
- Для реального бронирования подключается сторонний сервис / виджет через iframe или API.
- Архитектура должна предусматривать:
  - валидацию дат на клиенте
  - передачу данных на виджет/систему бронирования
  - fallback: строки контакта, если виджет недоступен.

## 9. АПИ и сторонние сервисы

- Карта: Google Maps / Yandex / OpenStreetMap.
- Отзывы: подключение виджета Google Reviews / Trustpilot / TripAdvisor.
- Соцсети: ссылки и мета-теги Open Graph.
- SEO: Schema.org для номеров, отеля и адреса.

## 10. Производительность и оптимизация

- Автоформатирование и сжатие изображений.
- Lazy-loading изображений и видео.
- Минимизация CSS/JS.
- Критический CSS для первого экрана.
- Предзагрузка ключевых изображений и шрифтов.

## 11. Доступность

- Семантический HTML.
- Текстовые альтернативы для всех изображений.
- Высокий контраст текста и кнопок.
- Фокусные стили для клавиатуры.
- Локализуемые метки и aria-атрибуты.

## 12. Разработка и тестирование

- Разработка на локальном сервере.
- Тестирование на мобильных и десктопах.
- Проверка скорости в Lighthouse.
- Проверка валидации HTML/CSS.

## 13. Этапы реализации

1. Настройка структуры проекта и шаблонов.
2. Разработка главной страницы и формы бронирования.
3. Создание страниц номеров и услуг.
4. Подключение галереи, отзывов и карты.
5. Оптимизация, тестирование, правки.
6. Запуск и сбор обратной связи.
